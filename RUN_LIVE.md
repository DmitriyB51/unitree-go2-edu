# Живая локализация на Go2 — как запустить руками

Проверено на живой собаке **2026-07-20**. Здесь ровно та последовательность, которая
сработала, плюс то, обо что мы спотыкались.

Схема простая:

```
СОБАКА (Jetson)                                    НОУТБУК
  Point-LIO        ─┐                                 RViz
  map-matcher       ├─ домен 0 (провод enP8p1s0)      домен 1 (WiFi wlo1)
  domain_bridge    ─┘ ───────── WiFi ────────────────────┘
```

**Почему так, а не проще:** приложение лидара Unitree — «голый» DDS и отдаёт данные,
только если CycloneDDS привязан к ОДНОМУ проводному интерфейсу. Поэтому Point-LIO и
матчер обязаны жить в домене 0 на проводе. Чтобы картинка попала на ноутбук по WiFi,
между доменами стоит `domain_bridge`.

**Тяжёлое облако `/registered_scan` (343 КБ/с) НЕ уходит по WiFi** — матчер работает
на самой собаке. По воздуху летят только поза, tf и лёгкое облако `aligned_cloud` (41 КБ/с).

---

## 0. Сеть

Собака и ноутбук — в одной сети. Рабочий вариант: телефонный хотспот **«Dima»**.

| | адрес |
|---|---|
| собака (WiFi) | `172.20.10.3` |
| ноутбук (WiFi) | `172.20.10.2` |
| собака (провод, запасной) | `192.168.123.18` |

Проверить:

```bash
ping -c3 172.20.10.3
ssh unitree@172.20.10.3 'echo ok'      # пароль sudo на собаке: 123
```

Если пользуешься **проводом**, подними профиль (он любит отваливаться сам):

```bash
nmcli con up go2-wired                  # даёт ноутбуку 192.168.123.222
```

> **NU_Guest не подойдёт.** Это открытая сеть с изоляцией клиентов: ноутбук и собака
> физически не смогут обмениваться пакетами, никакая настройка DDS это не лечит.

## 1. После перезагрузки Jetson — один раз

```bash
ssh unitree@172.20.10.3
echo 123 | sudo -S jetson_clocks        # частоты 883 МГц -> 1984 МГц
```

`jetson_clocks` **не переживает перезагрузку**, повторяй после каждой.

> ⛔ **Не пиновать Point-LIO на ядра и не давать ему realtime-приоритет.**
> `taskset -c 6,7` + `chrt -f 80` (файл `run_pointlio_rt.sh`) **ломает оценку**: робот стоит
> на месте, а его положение наматывает круги радиусом ~5 м с периодом ~25 с. Причина —
> `transform_everything` это Python-нода, кормящая IMU на 250 Гц; на общих 2 ядрах с тем же
> приоритетом она отдаёт IMU с джиттером, а Point-LIO работает **только по гироскопу** и
> входит в автоколебания. CPU при этом всего ~47 %, то есть узким местом он никогда не был.
> Используй обычный `run_pointlio.sh`.

## 2. Собака: Point-LIO

**Робот должен стоять на стартовой точке и НЕ двигаться** — при старте захватывается
начало отсчёта `camera_init` и калибруется IMU (~20 с).

```bash
ssh unitree@172.20.10.3
nohup setsid ~/run_pointlio.sh > ~/pointlio.log 2>&1 < /dev/null &
```

Дождись строки `IMU Initializing: 100.0%`:

```bash
tail -f ~/pointlio.log
```

---

## 3. Собака: матчер (локализация)

```bash
nohup setsid ~/run_matcher.sh > ~/matcher.log 2>&1 < /dev/null &
tail -f ~/matcher.log
```

Хорошие признаки в логе:

```
loaded 2058515 pts -> downsampled to 104921 (voxel 0.15 m)
published prior map (104921 pts) latched on /localization/map
captured floor reference: robot map-Z = ...
```

Карта берётся из `~/maps/loc_5_map.pcd` (путь прописан в
`~/dima_ws/install/go2_localization/share/go2_localization/config/localization.yaml`).

> Конфиг читается из `install/`, а не из `src/` — после правки нужен `colcon build`.

---

## 4. Собака: мост в WiFi

```bash
nohup setsid ~/run_loc_bridge.sh > ~/bridge.log 2>&1 < /dev/null &
```

Что он пробрасывает (`~/go2_loc_bridge.yaml`):

| направление | топик | зачем |
|---|---|---|
| собака → ноут | `/localization/pose` | красная стрелка (50 Гц) |
| собака → ноут | `/tf`, `/tf_static` | без них RViz ничего не разместит |
| собака → ноут | `/localization/map` | серая карта (latched, один раз) |
| собака → ноут | `/localization/aligned_cloud` | жёлтое облако (41 КБ/с) |
| собака → ноут | `/localization/fitness` | качество совпадения |
| **ноут → собака** | `/initialpose` | кнопка «2D Pose Estimate» |

Последняя строка — не мелочь: без обратного направления кнопка задания позы работать не будет.

---

## 5. Ноутбук: RViz

```bash
~/go2_mola_pipeline/scripts/run_rviz_wifi.sh
```

Должно появиться: серая карта, жёлтое живое облако, красная стрелка, синий след.

**Если жёлтое облако смещено относительно серой карты** — нажми **2D Pose Estimate**
и ткни туда, где робот стоит на самом деле. У нас это подтянуло fitness с 0.077 до 0.013.

---

## 6. Проверка, что всё живое

Мониторинг на собаке (пишет CSV раз в секунду):

```bash
nohup setsid ~/run_health.sh > ~/health.log 2>&1 < /dev/null &
tail -f ~/health_*.csv
```

Колонки: `t_wall, dist_m, dz_m, odom_hz, scan_hz, scan_pts, fitness, cpu_pct, load1`

| что смотреть | норма | тревога |
|---|---|---|
| `fitness` | 0.006–0.02 | > 0.3 — лок потерян; миллионы — GICP не нашёл соответствий |
| `dist_m` (стоя) | 0.3–0.4 м | качается на метры — Point-LIO колеблется |
| `dist_m` (в целом) | десятки м | сотни/тысячи — **разошёлся, только перезапуск** |
| `odom_hz` | ~700–1500 | 0 — Point-LIO упал |
| `scan_hz` | 15–16 | < 8 — проблема с лидаром |
| `cpu_pct` | ~47 % | — |

**Главный приём диагностики.** Плохой fitness бывает по двум разным причинам, и путать их
легко. Различает их `dist_m`:

| | робот не на стартовой точке | Point-LIO разошёлся |
|---|---|---|
| `dist_m` | нормальный (единицы–десятки м) | взлетает до сотен/тысяч |
| лечится | кнопкой «2D Pose Estimate» | только перезапуском Point-LIO |

Point-LIO вообще не знает про карту — это чистая одометрия от своей точки старта.
Поэтому «неправильно встал» на `dist_m` не влияет.

## 7. Остановить всё

```bash
ssh unitree@172.20.10.3 'pkill -f "[p]ointlio_mapping"; pkill -f "[t]ransform_everything"; pkill -f "[m]ap_matcher_node"; pkill -f "[d]omain_bridge"; pkill -f "[h]ealth_log"'
```

> Скобки `[p]` обязательны. Без них `pkill -f "pointlio_mapping"` совпадёт с самой
> ssh-командой и убьёт твою же сессию.

## Если что-то не работает

| Симптом | Причина | Что делать |
|---|---|---|
| RViz мгновенно падает, в логе `eno1: does not match an available interface` | отвалился проводной профиль | `nmcli con up go2-wired` |
| ноутбук не видит топиков | CycloneDDS ушёл не на тот интерфейс | проверь `env/cdds_laptop_wifi.xml`, имя через `ip -brief addr` |
| топики видно, но данных нет | мост упал | перезапусти `~/run_loc_bridge.sh` |
| робот стоит, а в RViz едет кругами | пиннинг ядер / realtime | запускай `run_pointlio.sh`, **не** `run_pointlio_rt.sh` |
| fitness миллионы, `converged=0` | Point-LIO разошёлся | перезапуск Point-LIO (робот стоит смирно) |
| ssh есть, но по проводу таймаут | кабель выдернули, когда двигали собаку | иди по WiFi на `172.20.10.3` |
